Що таке GNU/Linux

GNU/Linux — це поєднання ядра Linux (kernel) та набору утиліт GNU і додаткового програмного забезпечення. Це вільна, відкрита операційна система, яку можна модифікувати та розповсюджувати за відкритими ліцензіями (наприклад, GPL). Linux широко використовується на серверах, вбудованих системах, робочих станціях та хмарних середовищах.

Приклади базових команд CLI

Команди нижче — короткий набір для роботи з файловою системою та консоллю.

pwd
Показує поточний робочий каталог
echo "текст"
Виводить текст у консоль
ls -la
Перелік файлів каталогу з детальною інформацією
cd /path/to/dir
Зміна поточного каталогу
touch file.txt
Створює порожній файл або змінює час модифікації
mkdir newdir
Створює каталог
cp src dest
Копіює файл або каталог (-r для рекурсії)
mv old new
Перейменувати / перемістити
rm file
Видалення файлу (rm -r для каталогів)
history
Покажe історію введених команд

BASH-скрипт — імітація авторизації (текстовий режим)

Нижче — приклад простого shell-скрипта, який демонструє введення логіна/пароля і базову перевірку. Увага: у реальних умовах не зберігайте паролі у відкритому вигляді і використовуйте належні механізми аутентифікації.

# auth_demo.sh
#!/usr/bin/env bash

# Простий демонстраційний скрипт авторизації
# Збережіть цей файл як auth_demo.sh і зробіть виконуваним: chmod +x auth_demo.sh

# Жорстко вбудовані "користувачі" для демонстрації
declare -A USERS
USERS[alice]=password123
USERS[bob]=qwerty

read -p "Логін: " LOGIN
# -s приховує введення пароля
read -s -p "Пароль: " PASS
echo

# Невеликий таймаут для правдоподібності
sleep 0.6

if [[ -v USERS[$LOGIN] && "${USERS[$LOGIN]}" == "$PASS" ]]; then
  echo "Авторизація успішна. Ласкаво просимо, $LOGIN!"
  # Демонстраційні дані сесії
  echo "$(date) - Користувач $LOGIN увійшов(ла)"
  exit 0
else
  echo "Невірний логін або пароль. Спробуйте ще раз."
  # Лиміт простих спроб
  sleep 0.4
  exit 1
fi

Порада: щоб зробити більш реалістичну перевірку, замініть перевірку на роботу з хешами (наприклад, через sha256sum) або зберігайте користувачів у захищеному файлі з правами доступу.
