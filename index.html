Основи Git

Що таке системи контролю версій і Git

Системи контролю версій (VCS) дозволяють відстежувати зміни у файлах, зберігати історію проєкту та співпрацювати над кодом у команді. Git — це найпопулярніша розподілена система контролю версій, створена Лінусом Торвальдсом у 2005 році.

Основні можливості та переваги Git

  • Розподілена структура — кожен розробник має повну копію репозиторію.
  • Швидкість роботи з великими проєктами.
  • Гілкування (branching) та злиття (merging) для паралельної розробки.
  • Безпечне зберігання історії.
  • Інтеграція з популярними платформами (GitHub, GitLab, Bitbucket).

10 базових команд Git

  • git init – створює новий локальний репозиторій.
  • git clone <url> – копіює віддалений репозиторій на комп’ютер.
  • git status – показує стан файлів (змінені, додані).
  • git add <file> – додає файл у список для фіксації (staging area).
  • git commit -m "повідомлення" – зберігає зміни з коментарем.
  • git log – показує історію комітів.
  • git branch – відображає список гілок.
  • git checkout <branch> – перемикається на іншу гілку.
  • git merge <branch> – зливає гілку з поточною.
  • git push / git pull – надсилає або отримує зміни з віддаленого репозиторію.
